Descripción y Alcance del Proyecto

01 Gestión de Socios

Registro en línea de asociados con validaciones obligatorias (DNI único, domicilio, teléfono), automatizando la incorporación al Libro de Registro digital.

02 Módulo Transaccional

Historial completo de pagos de cuotas sociales mensuales y control de membresía con pasarela de pago recurrente mediante Mercado Pago.

03 Recaudación Transparente

Visualización pública de campañas de recaudación (insumos, obras y aparatología) con barras de progreso financiero calculadas en tiempo real.

04 Auditoría de Donaciones

Checkout interactivo para registrar transferencias bancarias manuales mediante carga de comprobantes, y donación directa en línea.

05 Panel Administrativo

Interfaz de control exclusiva para operadores para aprobar/rechazar transferencias, dar de alta/baja campañas y gestionar el padrón de socios.

06 Módulo Informativo

Portal dinámico para la publicación y difusión de novedades, obras concretadas y logros institucionales de la cooperadora del hospital.

Arquitectura de Software

Persistencia Híbrida y Modelo Cliente-Servidor

Cliente (Frontend SPA)
React.js · Vite · Tailwind CSS
⇅ HTTPS / REST API + JWT
API Server (Backend)
Node.js · Express · pnpm workspaces
SQL Transaccional ACID
  • `usuarios`: Credenciales con contraseñas hasheadas en `bcryptjs`.
  • `perfiles_socios`: Registro legal de asociados (DNI único).
  • `campanas_eco`: Balances y metas financieras de campañas.
NoSQL Documental Flexible
  • `noticias_actualidad`: Novedades con galerías y vídeos dinámicos.
  • `campanas_detalle`: Narrativa enriquecida vinculada por `campana_id_ref`.

⚡ Mecanismos Especiales de Datos:

Data Mashup: Para evitar latencias de red, el backend consulta en paralelo SQL y NoSQL mediante Promise.all y unifica los datos en una única respuesta estructurada para el cliente.
Evitación de Condiciones de Carrera: El endpoint de donación adquiere un bloqueo exclusivo sobre la fila SQL (SELECT ... FOR UPDATE) garantizando transacciones concurrentes seguras.

Patrones de Diseño y Refactorización

Análisis de Código y Propuestas de Refactorización

Singleton Pattern
Conexión de BD Única

Módulo: backend/config/db.js
Asegura una única instancia de conexión para los motores SQL y NoSQL, previniendo la fatiga del pool de conexiones.
Ventaja: Ahorro sustancial de memoria y control centralizado del estado de la base de datos.

class DB { static getInstance() { ... } }
Factory Method
Creación de Usuarios

Módulo: services/userService.js
Delegación de la creación de instancias del modelo según rol (Socio o Admin), precargando permisos específicos.
Ventaja: Cumplimiento del principio Open/Closed al extender tipos de usuarios sin alterar el controlador.

UserFactory.createUser(role, data);
State Pattern + Async Worker
Conciliación de Transferencias

Módulo: donacionController.js
El flujo de transferencias pasa a delegarse en estados: Pendiente, Aprobado o Rechazado, combinándose con un Worker asíncrono secundario para notificaciones interbancarias automáticas.
UX: Elimina el cuello de botella de aprobación manual.

donation.transitionTo(new ApprovedState());
